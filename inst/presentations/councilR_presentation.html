What is councilR?

Purpose & Overview

  • Collection of commonly used templates, color palettes, and functions
  • Developed by and for Metropolitan Council staff
  • Streamlines database connections and spatial data workflows

Database Connections

Database Overview

Seamless Data Access

  1. FRED - CD Research data (SQL Server)
  2. FRED-Oracle - Legacy Oracle database
  3. GISLibrary - Spatial datasets (ESRI SQL)

Security

Credentials managed via keyring package. See vignette("Credentials") for setup.

FRED Database

# Create connection
library(councilR)
fred <- fred_connection(db = "CD_RESEARCH_WEB", prod = FALSE)

# List available tables
DBI::dbListTables(fred)

# Import a specific table
gq_data <- import_from_FRED(table_name = "GQ_UNIT")

# Always disconnect!
DBI::dbDisconnect(fred)

Contains: Regional parks, population estimates, group quarters, and more

FRED-Oracle Database

Legacy Data Access

  • FRED_oracle_connection() - Connect to Oracle database
  • import_from_FRED_oracle() - Direct table import
  • Requires Oracle drivers and specific credentials
library(councilR)

# create connection
conn <- fred_oracle_connection()

# pull table using SQL
DBI::dbGetQuery(conn, "SELECT * FROM RESEARCH_WEB.RES_PERMIT_TYPE")

# disconnect
DBI::dbDisconnect(conn)

# import a specific table, with no additional SQL logic
import_from_FRED(table_name = "RESEARCH_WEB.RES_PERMIT_TYPE")

Contains: Some population estimates, housing data

GISLibrary Database

Spatial Data Repository

library(councilR)
# Connect to GISLibrary
gis <- gis_connection()

DBI::dbGetQuery(gis, "select *, Shape.STAsText() as wkt from GISLibrary.dbo.COUNTIES
                      where CO_NAME = 'ANOKA'") %>%
   st_as_sf(wkt = "wkt", crs = 26915)

# Import spatial tables as sf objects
counties <- import_from_gis(
  query = "GISLibrary.dbo.COUNTIES",
  .quiet = TRUE
)

# Non-spatial tables return data frames
lookup <- import_from_gis(
  query = "GISLibrary.dbo.CountyCTULookupTable",
  geometry = FALSE
)

import_from_gpkg()

  • import_from_gpkg() allows you to import spatial data from a GeoPackage file hosted on the Minnesota Geospatial Commons by providing a direct URL to the gpkg file.
  • Great for public facing code that needs to access spatial data without requiring users to access GISLibrary or other internal databases.
library(councilR)

# import the "TPP2050RegionallySignificantProjectPoints" layer only
import_gpkg("https://resources.gisdata.mn.gov/pub/gdrs/data/pub/us_mn_state_metc/trans_tpp2050/gpkg_trans_tpp2050.zip",
